VERSION NOTES

Changes made to this version:

  • Data enabled: Decision Support Server (DSS) uses data from the EHR to determine which clinical condition applies. DSS asks questions only if data is not already provided.
  • Contraindications: contraindication information can be passed in from the EHR, and DSS responds accordingly. If any specific contraindication is not mentioned in the incoming data, DSS assumes that there is no such contraindication for the patient.
  • DSS version 5.1.3, or later required.
  • Configuration of integration: for non-Epic integrations, such as Allscripts, set {skip.bpa.links} to "true", so DSS knows not to ask questions designed specifically for the Epic BPA screen. This is an optional "config item" in the domain deployment rule.
  • Configuration of advice text: set {advice.text.addendum} to a string of text to be appended to the advice text. For example, "If there are any questions on the most appropriate examination for your patient, please contact radiology consultation at ###-###-#### or abc@med.hospital.edu." This is an optional "config item" in the domain deployment rule.
  • Configuration of imaging context: if applicable, set {us.expertise.available} and/or {mr.techniques.available} to "true", so that this AUC can make recommendations where Ultrasound and/or MRI are appropriate. These are optional "config items" in the domain deployment rule.
  • Improvements to v4: changed some scores from 3 to 4; added missing logic related to cancer or infection condition
